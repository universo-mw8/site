FAQ

O que é o Startup DEV RUMBLE?

É uma competição de programação onde equipes de até 4 pessoas terão 48 horas para construir uma aplicação web.

Onde será realizado o concurso?

Virtualmente. Isto quer dizer que seu time pode trabalhar de qualquer lugar. Não é o futuro a internet?

Quando serão abertas as inscrições?

As datas estão divulgadas na página inicial da competição. Fique atento também no twitter da HE:labs.

Há limitação de vagas?

Tentaremos abrir a maior quantidade de vagas possíveis, mas teremos um espaço limitado, então lembre-se que os primeiros terão prioridade.

O que eu preciso saber para poder entrar na competição?

Conhecimento em alguma linguagem de programação para web. Também deve ter conhecimento em Git, para controle de versão do código.

Posso concorrer com uma aplicação mobile nativa para Android ou iOS?

Não. Somente são aceitas aplicações web.

Onde serão hospedados os repositórios Git?

Nós forneceremos um repositório Git privado para a competição. Você deve usar este repositório para o desenvolvimento e alguém do seu time deve integrar pelo menos 2 vezes por dia algum código lá para demonstrar o progresso.

Onde a aplicação será hospedada?

Cada equipe terá sua aplicação hospedada em um serviço fornecido pela organização.

Se minha empresa estiver patrocinando o Startup DEV RUMBLE, eu posso participar da competição?

Com certeza!

Eu posso usar gems, plugins, bibliotecas, templates, ícones, etc?

Sim. Nós encorajamos o uso de qualquer material que seja público a todos.

Posso desenvolver um plugin ou uma biblioteca para minha aplicação antes da competição iniciar?

Desde que o plugin ou biblioteca esteja disponível ao público e não forneca a maioria das funções do aplicativo. Seria aceitável para desenvolver e lançar um plugin que fornece uma interface para uma API, por exemplo. Seria inaceitável esconder seus controllers e views dentro de um plugin e "lançá-lo" apenas para uso na competição.

O que estou autorizado a fazer para me preparar para a competição?

Você é livre para planejar e projetar os requisitos para sua aplicação. Você também é livre para trabalhar em papel ou mockups da interface do usuário. Você não tem permissão para produzir material digital ou qualquer teste ou código do aplicativo antes do início da competição. Se você tiver dúvidas pode nos contatar.

Quando eu terei acesso ao repositório de código e ao serviço de hospedagem?

O acesso a ambos será liberado no início da competição.

A minha aplicação precisa ser open source?

Incentivamos as equipes a compartilhar seu código fonte, mas a competição não exige isso.

Posso dar push de algumas coisas antes do início da competição?

Não.

Posso mudar o nome do meu repositório?

Não.

Como posso obter mais informações?

Fique atento no twitter da HE:labs para as novidades e qualquer dúvida abra um chamado no Github.

Minha empresa ou organização gostaria de patrocinar o evento. Como posso fazer isso?

Entre em contato com os organizadores do concurso através do email contato@helabs.com.br.
